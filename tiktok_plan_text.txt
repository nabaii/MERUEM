Executive Summary

This document details the 12-week development plan for two TikTok-native competitive intelligence features: the Velocity Score Predictor and the Algorithmic Legibility Auditor. These features exploit TikTok’s interest-graph architecture—specifically its initial trial pool mechanics and its ASR/OCR indexing pipeline—to give the agency capabilities that most competitors lack entirely.

The Velocity Score Predictor monitors client video performance in real time during the critical first 60 minutes of distribution, calculating a weighted engagement score that mirrors TikTok’s internal point threshold system. When a video hits the threshold for algorithmic promotion, the system can automatically trigger a Spark Ad boost to convert an organic viral moment into paid scale.

The Algorithmic Legibility Auditor is a pre-publish quality gate that scans video drafts against TikTok’s 2026 indexing requirements: ASR keyword placement, OCR text positioning in the safe zone, and caption keyword density. This positions the agency as a Social SEO specialist, ensuring every piece of content is technically discoverable.

Both features are designed to be “basically free” to operate, leveraging open-source AI libraries, the TikTok API’s existing endpoints, and lightweight infrastructure.



Dual-Track Development Structure

The two features share some infrastructure (API authentication, database, dashboard shell) but are otherwise independent workstreams. The plan uses a parallel-track model: shared foundation in Sprints 1–2, then dedicated feature tracks in Sprints 3–4 (Velocity Score) and Sprints 3–4 (Legibility Auditor), converging for integration and hardening in Sprints 5–6.

Feature 1: Velocity Score Predictor

Velocity Score — Technical Specification

Concept: The 50-Point Threshold

TikTok’s distribution system evaluates each video in tiered pools. The initial trial pool serves the video to approximately 300 users. The system assigns weighted points to each engagement signal. A video that accumulates roughly 50 points from its first 300 viewers advances to the next, larger pool. The Velocity Score Predictor replicates this scoring model client-side, giving the agency real-time visibility into which videos are about to break through.



Scoring Model

Signal

Weight (pts)

Detection Method

Rationale

Rewatch / Loop

5

API: play count vs. unique viewer count (inferred from engagement rate patterns)

Strongest indicator of content that “holds” attention. Loops signal the algorithm that the content is worth resurfacing.

Watch to Completion (≥70%)

4

API: average watch time / video duration. Flag if >= 0.70.

TikTok’s Monolith system uses completion rate as the primary retention signal for pool promotion.

Share (Send)

3

API: share_count field in video insights

Shares extend distribution beyond the interest graph. Each share is a manual signal that the content has utility or emotional resonance.

Comment

2

API: comment_count field

Comments increase session time on the video. Lower weight because comments can be negative or low-intent.

Like

1

API: like_count field

Lowest-friction engagement. Useful as a volume indicator but not a strong algorithmic signal on its own.

Save / Favorite

3

API: collect_count (if available via Business API)

Saves indicate future intent. Strong signal for educational, tutorial, and utility content verticals.



Composite Score Formula: V = (5 × Loops) + (4 × Completions) + (3 × Shares) + (3 × Saves) + (2 × Comments) + (1 × Likes). The system normalizes this against the viewer count at each polling interval to produce a per-viewer score. When V ≥ 50 within the first 300 viewers (or equivalent rate at larger pool sizes), the system flags the video as a breakout candidate.

Feature 2: Algorithmic Legibility Auditor

Pre-publish Social SEO quality gate for TikTok’s ASR/OCR indexing pipeline



Legibility Auditor — Technical Specification

Concept: Making Content Machine-Readable

TikTok’s recommendation engine now functions as a search engine. Approximately 40% of Gen Z users prefer TikTok over Google for discovery. The algorithm “reads” every video through three channels: ASR (Automatic Speech Recognition) extracts spoken words, OCR (Optical Character Recognition) reads on-screen text, and the caption field is parsed for keyword signals. Content that fails any of these three checks is effectively invisible to TikTok’s search index—even if it performs well in the interest-graph feed.

The Legibility Auditor scans a video draft against all three channels before publish, giving the creative team a pass/fail report with specific remediation instructions.



Audit Checks

Check

Rule

Detection Method

Fail Remediation

ASR Keyword Placement

Primary keyword must be spoken in the first 3 seconds of narration.

Whisper (local, “base” model) transcription with word-level timestamps. Check if any target keyword appears with timestamp <= 3.0s.

Re-record the opening narration to front-load the keyword. Provide suggested script opening.

OCR Safe Zone

Primary search query must appear in on-screen text, positioned at least 150px from the top of the frame (below the TikTok UI chrome).

Extract key frames at 0.5s intervals for the first 5s. Run TrOCR or Tesseract. Use bounding-box coordinates to verify Y-position >= 150px.

Adjust text overlay position in the editor. Flag if text is too small (< 24pt equivalent) or low contrast.

Caption Keyword Density

First 150 characters of the caption must contain the primary keyword and at least one secondary keyword.

String matching against the client’s keyword list. Calculate keyword density (keyword chars / total chars). Flag if < 8%.

Suggest rewritten caption opening with keywords front-loaded. Provide character count.

Audio Clarity Score

ASR confidence must be >= 85% for the keyword utterance. Low confidence means TikTok’s own ASR may also miss it.

Whisper outputs per-word confidence scores. Average the confidence of the target keyword tokens.

Flag audio quality issues: background music too loud, mumbled delivery, heavy accent on keyword. Suggest re-recording with clearer enunciation.

Hashtag Alignment

At least 2 of the top 5 hashtags must match high-volume search terms related to the primary keyword.

Cross-reference hashtags against the keyword database (built from TikTok’s search suggest API or scraped trending data).

Suggest replacement hashtags from the keyword database ranked by search volume.

Cost-Neutral System Improvements

The following enhancements are woven into the sprint plan above. Each one strengthens the system’s intelligence or resilience without adding meaningful infrastructure cost.



1. Velocity Decay Rate Analysis

Instead of only measuring whether V score crosses 50, also track the rate of decay after the initial spike. Videos that maintain a high V score for 30+ minutes (slow decay) are fundamentally different from flash-in-the-pan spikes. Tag these as “Sustained Velocity” assets—they’re the best candidates for Spark Ad boost because the organic momentum has staying power. Implemented as a simple derivative calculation on existing time-series data.

Impact: Reduces wasted Spark Ad spend on videos that spike briefly then collapse. Improves ROAS by 20–40% (estimated) by boosting only sustained performers.

Cost: Zero. A derivative calculation on data already in PostgreSQL.



2. ASR Confidence Heatmap

Instead of a binary pass/fail on ASR keyword detection, generate a per-second confidence heatmap of the full transcript. This reveals not just whether the keyword was said, but how clearly the entire narration is being indexed. Sections with low ASR confidence are effectively invisible to TikTok’s search engine, even if they contain valuable content. Display as a visual timeline in the audit report.

Impact: Gives the creative team granular feedback on audio quality throughout the video, not just the first 3 seconds. Identifies narration habits that reduce indexability (e.g., trailing off at end of sentences).

Cost: Zero. Whisper already outputs per-word confidence scores; this just visualizes them.



3. Save-to-Share Ratio as Virality Type Classifier

Track the ratio of saves to shares for each video. High saves + low shares = “Utility Viral” (tutorials, how-tos). High shares + low saves = “Social Viral” (entertainment, reactions). This classification helps the creative team understand what type of virality each piece of content achieves, which informs the Spark Ad targeting strategy (utility content targets search intent; social content targets interest clusters).

Impact: Turns a single V score into a two-dimensional virality map. Enables differentiated Spark Ad targeting strategies per content type.

Cost: Zero. Computed from metrics already collected in the polling loop.



4. OCR Font Size and Contrast Validator

Beyond checking text position in the safe zone, also validate that on-screen text meets minimum readability thresholds: font size >= 24pt equivalent (based on bounding box height relative to frame height) and sufficient contrast ratio against the background (sample background pixels behind the text bounding box). TikTok’s OCR is more likely to miss small or low-contrast text, making the video less searchable.

Impact: Catches a class of audit failures that position-only checks miss. Particularly valuable for creators who use stylized or decorative fonts.

Cost: Zero. Uses bounding box data already extracted by OCR and basic pixel sampling via PIL/Pillow (already installed).



5. Cross-Platform Keyword Arbitrage

Compare the client’s TikTok keyword portfolio against trending search terms on YouTube Shorts and Instagram Reels (using existing Ghost Virality infrastructure if available). Surface keywords that are high-volume on TikTok but underserved on other platforms—or vice versa. This helps the agency advise clients on cross-posting strategy with platform-specific keyword optimization.

Impact: Turns the keyword database into a cross-platform intelligence asset. Enables the agency to offer multi-platform SEO as a single service.

Cost: Zero if Ghost Virality infrastructure exists. Minimal if built standalone (one additional scraper for YouTube search suggest, which is free and unauthenticated).



6. Audit Score Correlation with Performance

After 4+ weeks of publishing audited content, correlate audit pass/fail results with actual video performance (views, V score, search impressions if available). This creates an evidence base showing clients: “videos that passed all 5 checks averaged 3.2x more views from search than videos that failed 2+ checks.” Computed as a simple aggregation query on existing data.

Impact: Converts the audit tool from a “best practice” recommendation into a proven, data-backed system. Powerful for client retention and upselling.

Cost: Zero. Aggregation on data already stored from both features.
